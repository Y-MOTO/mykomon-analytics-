## 1. これまでに構築したシステムの全体像

今回、3つのツールを一体で開発しました。

### フェーズ1｜日報入力補助ツール（Microsoft Edge 拡張機能）

職員がMyKomonで日報を入力する際、画面右上に小さなパネルが表示されます。職員はそこから

- **業務区分**（申告／相談／監査／記帳代行 など）
- **ステータス**（完了／継続／中断）
- **工程**（資料回収中／作業中／チェック中 など）
- **阻害要因**（資料待ち／確認中／客先都合 など）
- **緊急度**（Red／Yellow／Green）
- **工数**（〜1時間／〜2時間 など）

を選んで「平文に反映する」ボタンを押すと、以下のようなタグが日報末尾に自動挿入されます。

```
---
【業務区分】監査
【ステータス】継続
【工程】チェック中
【阻害要因】資料未回収（顧客）
【緊急度】Yellow
【工数】〜2時間
#業務区分_監査 #ステータス_継続 #工程_チェック中 #阻害要因_資料未回収（顧客） #緊急度_Yellow #工数_〜2時間
```

職員はこのタグの上に自由記述を加えて、MyKomonの「登録する」ボタンで保存します。

**タグ入力を促す警告機能（2026年4月追加）**

タグを入力せずに「登録する」ボタンを押すと、以下の警告ダイアログが自動表示されます。

```
⚠️ タグが入力されていません
日報補助ボタンから業務区分・ステータスを入力すると、
後から集計・AI分析に活用できます。

[ タグを入力する ]  [ このまま登録する ]
```

- **「タグを入力する」**：補助パネルが開き、選択してから登録できます。
- **「このまま登録する」**：緊急時など意図的にタグなしで登録する場合に使用します。

完全なブロックではなく「意図的な操作が必要な抵抗」として機能します。

### フェーズ2｜CSV自動取得ツール

MyKomonには外部APIがないため、IDとパスワードを使って自動的に日報CSVを取得する仕組みを組み込みました。分析アプリのサイドバーから期間を指定してボタンを押すだけで、最新の日報データを取り込めます。

**【重要】同じ期間を複数回エクスポートした場合の動作**

同じ月のデータを再度エクスポートしても、データが二重に蓄積されることはありません。月ごとにCSVファイルが1つ作成される仕組みのため、再エクスポートすると既存のファイルが新しいデータで上書きされます。

- **新しい月のデータ** → 新規ファイルとして追加
- **すでにエクスポート済みの月** → 上書き（最新データに更新）

このため、月次レビューのたびに同じ期間を含めて取得し直しても問題ありません。直近月の日報が増えた場合も、再エクスポートするだけで最新の状態に更新されます。

### フェーズ3｜日報分析Webアプリ（ローカル版 ＆ クラウド版）

取得したCSVをAIが読み込み、経営者視点で分析レポートを生成します。

| 版 | 使い方 | 用途 |
|---|---|---|
| **ローカル版** | 所長PCでバッチをダブルクリック → `http://localhost:8501` | 所内PCで使用 |
| **クラウド版（推奨）** | `https://esgn4cndlvm38hucszjpes.streamlit.app` | 外出先・スマホからも利用可 |

> **クラウド版は日報分析＋人事経営相談が1つのURLに統合されています。**  
> MyKomon IDとパスワードを入力するだけで、CSVエクスポートから人事経営相談まで一気通貫で使えます。

## 2. 職員へのアプローチ（拡張機能の導入）

分析アプリを活用するには、**職員が日報にタグを入れるデータが蓄積されること**が前提です。以下の手順で導入してください。

### ステップ1｜拡張機能のインストール（IT管理者が対応）

1. ZIPファイルと `install.bat` を各職員のPCに配布する
2. 職員は `install.bat` をダブルクリックするだけでインストール完了
3. Edgeの右上に小さなアイコンが表示されれば完了

### ステップ2｜職員への説明（10分程度の朝礼で可）

伝えるべきことは3点だけです。

> 「MyKomonで日報を書くとき、画面右上にパネルが出ます。業務区分・ステータス・阻害要因を選んで『平文に反映する』を押してから、登録してください。選択は1〜2分で終わります。自由記述は今まで通り書いてください。」

### ステップ3｜定着のポイント

- **最初の1ヶ月は「完了/継続/中断」の3択だけ**でも十分です。全項目を一度に求めると負担になります。
- タグなしで「登録する」を押すと警告ダイアログが表示されるようになりました。完全なブロックではありませんが、「うっかり忘れ」を防ぐ抑止力として機能します。
- 週1回、分析アプリの**「担当者別 タグ付き率グラフ」**を確認し、使用率が低い職員に個別に声かけをすると定着が早まります。
- 週1回、所長が日報の一部を読んで「タグがついていた」ことに言及するだけで定着率が上がります。

## 3. タグ付きデータが生成される仕組み

職員が拡張機能を使って日報を登録すると、MyKomonのデータベース上の日報テキストに構造化タグが含まれた状態で保存されます。

```
○○株式会社の月次監査を実施。試算表は概ね完了したが棚卸表が
未着のため確定できず。

---
【業務区分】監査
【ステータス】中断
【阻害要因】資料待ち
#業務区分_監査 #ステータス_中断 #阻害要因_資料待ち
```

このタグは**MyKomonが通常どおり保存する平文テキストの一部**です。特別な連携や追加設定は不要で、翌月にCSVエクスポートした段階で自動的に分析アプリが読み取ります。

タグのない過去の日報も「全日報」として件数・工数の集計対象になります。**タグ付き日報が増えるほど、業務区分・詰まり要因・担当者別の深い分析が可能になります。**

## 4. クラウド版分析アプリの使い方

### 基本的な使い方（月次レビュー想定）

**① データを取得する**

左サイドバーの「MyKomon ID」「パスワード」に事務所のMyKomonログイン情報を入力し、期間（例：2026年1月〜3月）を選んで「CSVをエクスポート実行」を押します。数十秒で日報データが読み込まれます。

**② ダッシュボードを確認する**

画面上部のタブで切り替えます。

| タブ | 内容 |
|---|---|
| 担当者別分析 | 職員ごとの日報件数・**タグ付き率（使用率）**・詰まり件数 |
| 顧問先分析 | 顧客ごとの詰まり件数・阻害要因の傾向 |
| 業務・工程ボトルネック | どの業務区分・工程で詰まりやすいか |
| 月次トレンド | 月ごとの処理能力・詰まり率の変化 |

**担当者別 タグ付き率グラフ（2026年4月追加）**

「担当者別分析」タブに、職員ごとの拡張機能使用率を示す棒グラフが追加されました。  
色が赤いほど未使用（タグなし日報が多い）、緑に近いほど適切に使用されています。  
月次レビュー時に確認し、使用率が低い職員への個別フォローの指標として活用してください。

**③ AIレポートを生成する**

「ai分析レポート」タブを開き、「📝 AIレポートを生成」ボタンを押します（30〜60秒）。AIが以下の3部構成でレポートを出力します。

1. **経営者への要約**（全体の状況を3〜5行で）
2. **要注意事項**（優先順位付きの箇条書き）
3. **推奨アクション**（担当者名・顧問先名を含む具体的な指示）

**④ 印刷・共有**

「🖨️ このレポートを印刷」ボタンで、レポートのみを印刷用HTMLで開きます。月次経営会議の資料として印刷・配布が可能です。

**⑤ フォローアップ質問**

レポート下部の質問欄に追加質問を入力できます。例：

> 「○○社の詰まりが長期化している原因を詳しく分析してください」

## 6. 配布フォルダと必要なインストール作業

今回の開発成果物は、以下の3つのフォルダに整理されています。

| フォルダ | 配布対象 | 内容 |
|---|---|---|
| **職員配布用** | 全職員 | `install.bat`（Edge拡張機能インストーラー） |
| **所長配布用** | 所長 | クラウド版起動・ローカル版起動・運用説明書 |
| **IT管理者用** | IT管理者のみ | APIキー暗号化ツール（取扱い注意） |

### クラウド版だけを使う場合はIT管理者用フォルダは不要です

クラウド版のAPIキーは既にStreamlit Cloud Secretsに登録済みです。所長配布用フォルダの「**【クラウド版】起動.bat**」をダブルクリックするだけで、APIキーの設定なしにAI分析レポートまで全機能が使えます。

IT管理者用フォルダが必要になるのは、**ローカル版でAI分析レポートを使う場合の初回のみ**です。

### 用途別の必要フォルダ

| 用途 | 職員配布用 | 所長配布用 | IT管理者用 |
|---|---|---|---|
| 職員が日報タグ入力 | ✅ 必要 | — | — |
| 所長がクラウド版で分析 | — | ✅ 必要 | **不要** |
| 所長がローカル版でも分析 | — | ✅ 必要 | ✅ 初回のみ必要 |

### IT管理者用フォルダをDropboxや共有フォルダに置いてはいけない理由

IT管理者用フォルダ内の `【管理者用・DROPBOX管理不可】APIキー暗号化.bat` は、実行前にAPIキーを平文でファイル内に書き込む仕組みです。Dropboxや共有フォルダに置くと、書き込んだAPIキーがそのまま同期・共有されてしまい、第三者に漏洩するリスクがあります。APIキーはクレジットカードに相当するものであり、漏洩すると不正利用による高額請求が発生する可能性があります。

**正しい手順：**
1. IT管理者がオフラインのPCでAPIキーを書き込む
2. USBで所長PCに持参して実行
3. 実行後すぐにbatファイルをUSBからも所長PCからも削除

## 7. 人事経営相談AIシステム

### 概要

所長が「計画と現実のずれ」や「経営者としての人事の悩み」を入力すると、Claude AIが根本原因・改善策・世界の事例を提案する独立したアプリです。

日報分析アプリ（データで「現状を見る」）と人事経営相談アプリ（ずれを「相談して対策を得る」）はセットで活用することを推奨します。

**相談できる内容の範囲（2種類）**

| 種類 | 例 |
|---|---|
| 人事制度改革のずれ | タグ付き率が上がらない・評価制度への不満・OKRの形骸化 |
| 経営者の人事一般相談 | G8各国の評価制度比較・Z世代の動機づけ・心理的安全性・採用戦略 |

### 起動方法

**クラウド版（推奨）**：`https://esgn4cndlvm38hucszjpes.streamlit.app` を開き、サイドバーで「💼 人事経営相談」を選択。

**ローカル版**：`【ローカル版】【人事経営相談】起動.bat` をダブルクリック → `http://localhost:8502`  
（日報分析アプリ・8501番と同時起動可）

### AIが参照している知識ベース（knowledge.py）

人事経営相談AIは以下の情報を踏まえた上で回答します。

| 情報 | 内容 |
|---|---|
| 会社プロフィール | 社員55名・有資格者7名・クライアント600先 |
| 人事制度改革の基本方針 | ジョブ型＋集合ボーナスモデル |
| 職階設計（Tier1〜4） | 各Tierの役割・評価KPI |
| 評価指標 | 定量40%・定性40%・組織貢献20% |
| 移行ロードマップ | 3フェーズ・18ヶ月の計画 |
| 失敗パターン | ボトムアップ失敗の6類型・成果主義失敗事例 |
| G8各国の事例（基本） | 米英独仏伊等の成功・失敗事例 |
| G8各国の詳細比較 | 各国の雇用形態・給与制度・評価制度・経営者視点を国別に詳述 |
| 経営者の人事一般知識 | 離職防止・採用・心理的安全性・世代間マネジメント・公平性 |
| 想定リスクと対策 | 6項目のリスクと対処方法 |

**knowledge.py の内容を更新することで、AIへの指示（知識ベース）を随時更新できます。**  
制度が変更になった場合や新たな課題が判明した場合は、開発担当者にお知らせください。

### サイドバーの機能

| セクション | 機能 |
|---|---|
| 📑 人事制度改革資料 | 詳細解説書・役員向けプレゼンの開く／印刷（Wordアプリ・PowerPointが起動） |
| 📖 使用マニュアル | 人事経営相談アプリ専用マニュアルをブラウザで表示・印刷 |
| 📊 日報分析データを使う | 日報分析アプリから送られた集計数値を相談入力欄に自動展開 |
| 📂 相談カテゴリ | 相談テーマの選択（13種類。G8事例研究・人事一般相談も含む） |
| 📋 現在の計画概要 | Phase 1〜3の概要確認 |

### 日報分析データとの連携

日報分析アプリの集計結果を「ズレ」として人事経営相談に自動転記できます。

**操作手順（クラウド版）：**

1. 「📊 日報分析」ページ → タブ5「AI分析レポート」でレポートを生成
2. 「**📨 人事経営相談に送る**」ボタンを押す
3. サイドバーで「**💼 人事経営相談**」に切り替え
4. サイドバーの「📊 日報分析データを使う」→「**📥 この結果を相談に使う**」をクリック
5. 入力欄に日報データが自動展開 → 補足を追記して「相談する」を押す

**操作手順（ローカル版）：**

1. 日報分析アプリ（8501）タブ5「AI分析レポート」でレポートを生成
2. 「**📨 人事経営相談に送る**」ボタンを押す（`latest_gap.json` に保存）
3. 人事経営相談アプリ（8502）のサイドバー「📊 日報分析データを使う」で数値を確認
4. 「**📥 この結果を相談に使う**」で入力欄に自動展開
5. 所長が補足を追記して「相談する」を押す

転記される内容：全件数・タグ付き率・詰まり件数・主な阻害要因・タグ付き率が低い担当者・緊急顧問先・月次詰まり率

### 相談履歴の保存

相談内容と回答は `consult_history.json` に自動保存されます。  
月次ミーティングや制度の見直し時の参考資料として活用できます。

画面右側の履歴パネルから過去の相談を確認でき、JSONファイルとしてダウンロードも可能です。

### 推奨される活用シーン

```
【月次サイクル】
日報分析アプリでCSVを取得・ダッシュボード確認
        ↓
タブ5「AIレポートを生成」→「📨 人事経営相談に送る」
        ↓
人事経営相談アプリ「📥 この結果を相談に使う」
        ↓
所長が補足を追記して「相談する」（10〜30秒）
        ↓
AIの回答をもとに翌月の対策を決定
        ↓
次の月次レポートで効果を確認（ループ）

【随時相談】
G8各国の人事制度を知りたい・Z世代への対応を相談したい
        ↓
カテゴリ「G8・海外事例の研究」または「経営者の人事一般相談」を選択
        ↓
相談例ボタンまたは自由入力で相談する
```

## 8. 今後の拡張について

現在のシステムは**最小限の項目で素早く始める**ことを優先して設計しています。運用が定着した後、以下の拡張が可能です。

- 業務区分の選択肢を5種類から18種類（税目別の詳細区分）へ拡充
- 阻害要因の選択肢をより詳細な15種類へ拡充
- 難易度・自己評価・関与税理士の記録欄の追加

ただし、**一度運用を開始したタグの名称は変更しないことが原則**です。変更すると過去データとの整合性が失われます。拡充は選択肢の「追加」のみ行います。
