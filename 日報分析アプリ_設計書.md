# 日報分析Webアプリ 設計書

## 概要

フェーズ2（CSV自動エクスポート）でネットワークドライブに蓄積された日報CSVを読み込み、
所長が経営者視点で業務状況を分析・把握するための Streamlit Webアプリ。

---

## 確定仕様

| 項目 | 内容 |
|------|------|
| 形式 | Streamlit Webアプリ（ブラウザで操作） |
| 操作者 | 所長が自分でブラウザを開いて使用 |
| 出力 | 画面表示のみ |
| CSV読み込み | ネットワークドライブを自動読み込み |
| 起動方法 | 起動.bat をダブルクリック |

---

## 全体アーキテクチャ

```
MyKomon（日報入力）
    ↓ Edge拡張機能で構造化タグを挿入
ネットワークドライブ（CSVが蓄積される）
    ↑ フェーズ2バッチが定期的に自動保存
    ↓ 分析アプリが自動読み込み
Streamlit Webアプリ（所長が分析）
    ↓ Claude API へ分析依頼
経営コメント・課題提言
```

---

## config.json 共有設計

フェーズ2のバッチセットアップ時にネットワークドライブのパスを **1回だけ入力** すれば、
CSV自動エクスポートと分析アプリの両方がそのパスを参照する。

```json
{
  "save_path": "\\\\server\\nippo",
  "export_days": 30
}
```

- **CSV自動エクスポートバッチ**: このパスにCSVを書き込む
- **日報分析アプリ**: このパスからCSVを読み込む
- 設定の二重管理が不要

## 分析画面の構成（予定）

### ダッシュボード（トップページ）

- 読み込んだCSVの期間・件数の表示
- 全体の完了率・中断率（サマリー）

### 分析パネル①：担当者別パフォーマンス

- 担当者ごとの完了・継続・中断の件数
- 「詰まっている人」の可視化

### 分析パネル②：阻害要因ランキング

- 資料待ち・確認中・知識不足・客先都合などの件数
- 棒グラフで表示

### 分析パネル③：顧客別リスク状況

- 中断・継続件数の多い顧客一覧
- 問題顧客の特定に活用

### 分析パネル④：業務区分別分布

- 監査・申告・記帳代行・相談などの割合
- 円グラフまたは棒グラフ

### 分析パネル⑤：月次トレンド

- 完了率の月次推移
- 中断件数の増減傾向

### 分析パネル⑥：AIによる経営コメント（Claude API）

- 上記データを元に Claude API が経営者向けに文章で要約
- 「今月の課題」「対処が必要な担当者・顧客」「推奨アクション」を出力

## プロンプト設計方針（Claude API）

分析の視点を明確に指定することで、的確な経営コメントを引き出す。

### プロンプトの基本構造

```
あなたは税理士事務所の経営管理コンサルタントです。
以下の日報データ（集計済み）をもとに、所長向けの経営レポートを作成してください。

## 分析データ
{集計結果をJSON形式で渡す}

## 出力形式
1. 今月の業務状況サマリー（3行以内）
2. 注意が必要な担当者・顧客（具体名で）
3. 主な阻害要因と対策提言
4. 来月に向けた優先アクション（3項目）
```

### プロンプト設計の注意点

- 「誰の立場で」→ 税理士事務所の経営者・所長
- 「何のために」→ 業務改善・人員配置の判断材料
- 「どう使うか」→ 月次ミーティングや個別指導の根拠として
