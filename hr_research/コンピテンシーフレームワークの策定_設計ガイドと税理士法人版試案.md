## 1. コンピテンシーフレームワークとは何か

### 1-1. 定義

> 「組織において高い成果を上げるために必要な能力・スキル・行動特性を、職階・役割別に体系化した共通言語」

**重要：** 「評価表」ではない。人事制度と教育制度の両方を動かす**設計図**。

### 1-2. フレームワークがない組織で起きること

| 問題 | 根本原因 |
|---|---|
| 「なぜあの人が昇格したのか分からない」 | 昇格基準が暗黙知・属人的 |
| 「何を頑張ればいいか分からない」 | 次の等級への到達像が不明 |
| 「研修が業務に役立っていない」 | 育成目標と評価基準が連動していない |
| 「評価に納得できない」 | 評価者によって基準がバラバラ |
| 「優秀な人の知恵が継承されない」 | 何が「優秀」かを言語化していない |

### 1-3. フレームワークがある組織で実現すること

- **採用：** 「どんな人を採るか」の共通基準ができる
- **評価：** 「何ができているか」を観察可能な行動で判断できる
- **昇格：** 「次に何が必要か」を全員が知っている
- **育成：** 「今期の育成目標」が評価基準から自動的に導かれる
- **処遇：** 「なぜその給与なのか」の根拠ができる

**数値根拠：**
- 多段階フレームワーク導入で昇格基準の明確度が **27%向上**（Gartner）
- 明確な習熟度レベルを持つ組織は内部異動率が **24%高い**（Gartner）

## 2. 国際標準フレームワークの構造

### 2-1. CGMA コンピテンシーフレームワーク（AICPA・CIMA）

会計・財務専門職向けの国際標準。**4領域 × 4習熟度レベル**で構成。

#### 4つの領域（ドメイン）

| ドメイン | 内容 | 初級での比重 | 上級での比重 |
|---|---|---|---|
| **Technical（技術）** | 会計・税務・財務の専門知識 | 高 | 低 |
| **Business（ビジネス）** | 経営・戦略・業界知識 | 中 | 中 |
| **People（人・関係）** | コミュニケーション・チームワーク・育成 | 低 | 中 |
| **Leadership（リーダーシップ）** | 変革・意思決定・組織貢献 | 低 | 高 |

→ **キャリアが進むにつれて「技術」から「リーダーシップ」へ比重がシフトする**設計が本質。

#### 4つの習熟度レベル

| レベル | 定義 | 相当する職階 |
|---|---|---|
| **Foundational（基礎）** | 基本的な理解。自分の行動で成果を出す段階 | スタッフ |
| **Intermediate（中級）** | 組織全体への理解が広がる。限定的に他者への責任を持つ | シニアスタッフ |
| **Advanced（上級）** | 強い分析力と戦略的理解。チームを通じて成果を出す | マネージャー |
| **Expert（熟達）** | 組織全体のリーダー。方向性を設定し変革を主導する | シニアマネージャー/パートナー |

### 2-2. AICPA CPA Firm Competency Model

米国公認会計士協会が設計した事務所向け標準モデル。

**5つの職階定義：**

```
Associate（新人）
   ↓ 2〜3年
Senior（シニア）
   ↓ 2〜4年
Manager（マネージャー）
   ↓ 3〜5年
Senior Manager / Director（シニアマネージャー）
   ↓ 3〜5年
Partner（パートナー）
```

**5つのコアコンピテンシー：**
1. 専門的知識・技術スキル
2. クライアントサービス・関係構築
3. リーダーシップ・マネジメント
4. ビジネス開発・戦略思考
5. プロフェッショナリズム・倫理観

各職階 × 各コンピテンシーに「この等級ではこの行動が見られること」を定義したマトリクスが核心。

## 3. コンピテンシーフレームワーク設計の7ステップ

### Step 1：組織の戦略・価値観を確認する（1〜2週間）

フレームワークは「組織が目指す姿」から逆算して作る。先に戦略を確認しないと、業界標準の丸写しになる。

**確認事項：**
- 3〜5年後に自社がどんな事務所でありたいか
- どんなスタッフが増えてほしいか
- 今「組織として足りていない」と感じる能力は何か

### Step 2：職階（グレード）を定義する（1週間）

フレームワークの「縦軸」を決める。

**設計の原則：**
- 段階数は5〜6が標準（少なすぎると粗く、多すぎると運用しにくい）
- 「スタッフ・シニア・マネージャー・シニアマネージャー・パートナー」が会計専門職の世界標準
- 各職階の**在籍年数の目安**と**典型的な業務範囲**をセットで定義する

### Step 3：コンピテンシー領域を選定する（1〜2週間）

フレームワークの「横軸」を決める。

**設計の原則：**
- 領域数は4〜6が最適（少なすぎると粗く、多すぎると評価が煩雑）
- 業種・組織に合わせてカスタマイズする（丸写しは機能しない）
- 全員に共通する「コア領域」＋役職に応じた「専門領域」の二層構造が使いやすい

### Step 4：ハイパフォーマーのインタビューと行動抽出（2〜4週間）

**最重要ステップ。** 優秀な実在の社員が「何をやっているか」を観察・インタビューして行動指標を作る。

**方法：**
- 各職階で「この人は確かに優秀だ」と全員が認める2〜3名を選ぶ
- 「最近うまくいった仕事の話を聞かせてください」→ 具体的行動を抽出
- 「その仕事でどういう判断をしましたか？」→ 思考パターンを抽出
- 抽出した行動を領域ごとに整理し、「この等級ではこう動く」の言葉に変換する

### Step 5：行動指標（Behavioral Indicator）の執筆（2〜3週間）

各セル（職階 × コンピテンシー）に「観察可能な具体的行動」を記述する。

**良い行動指標の条件：**
- **観察できる**：「態度が良い」ではなく「ミスを指摘されたとき、言い訳をせず改善策を提案する」
- **具体的**：「コミュニケーション力がある」ではなく「顧客の質問に対して24時間以内に回答する」
- **等級間で段差がある**：「指示を受けてやる」→「自分で判断してやる」→「他者に指示してやる」

**NG例 vs OK例：**

| NG（抽象的） | OK（具体的・観察可能） |
|---|---|
| コミュニケーション能力が高い | 顧客からの問い合わせに1営業日以内に回答し、不明点は「確認します」と伝える |
| 責任感がある | 期限を守れないと分かった時点で上司に先に報告し、対策案を一緒に提示する |
| リーダーシップがある | チームの仕事の進捗を週1回確認し、遅れているメンバーに声をかけて障害を取り除く |
| 成長意欲がある | 月に1冊以上の専門書を読み、学んだ内容を業務に1つ以上適用しようとする |

### Step 6：ドラフトのフィードバックと検証（2〜3週間）

作成したドラフトを関係者に見せ、修正する。

**フィードバックの取り方：**
- 各職階の代表者3〜5名に「自分のことが書かれているか？」「上の等級を見てどう思うか？」を聞く
- 管理職に「この基準で評価できるか？」を確認する
- 「この行動指標を見ても、誰が書いたか分からない（属人化していない）」になるまで修正する

### Step 7：実装・運用・定期更新

- **90〜180日の実装計画**を立て、段階的に導入する
- Phase 1：管理職への説明・研修
- Phase 2：1〜2部門でパイロット運用
- Phase 3：全社展開
- **年1回の見直し**：事業環境・採用基準・育成目標の変化に合わせて更新する

## 4. 税理士法人版コンピテンシーフレームワーク【試案】

### 4-1. 職階定義（5段階）

| Grade | 職階名 | 在籍目安 | 代表的な役割 |
|---|---|---|---|
| G1 | スタッフ | 0〜3年 | 指示のもと、標準的業務を正確にこなす |
| G2 | シニアスタッフ | 3〜6年 | 担当案件を自律的に進め、後輩のサポートをする |
| G3 | 主任 | 6〜10年 | 複数案件を管理し、G1・G2を指導する |
| G4 | マネージャー | 10〜15年 | チームを束ね、顧客との主担当責任者になる |
| G5 | シニアマネージャー | 15年〜 | 組織の方向性に関与し、所長を補佐する |

### 4-2. コンピテンシー領域（5領域）

| 領域 | 内容 |
|---|---|
| **①専門知識・技術** | 税務・会計・法令の知識と実務への適用力 |
| **②業務遂行** | 品質・期限・段取り・問題解決の実行力 |
| **③顧客・対外関係** | 顧客対応・コミュニケーション・信頼構築 |
| **④チーム・組織貢献** | 情報共有・育成支援・チームワーク |
| **⑤変革・成長志向** | 自己研鑽・改善提案・リーダーシップ |

### 4-3. 行動指標マトリクス（試案）

#### 領域①：専門知識・技術

| Grade | 行動指標（この等級ではこれができる） |
|---|---|
| G1 | 上司の指示のもと、基本的な税務申告書・決算書の作成ができる。不明点を都度確認する |
| G2 | 担当業種の一般的な税務・会計処理を自力で判断できる。税理士試験科目を1科目以上合格している |
| G3 | 複雑な税務案件（組織再編・事業承継等）について上司に意見を述べられる。法改正の影響を自ら確認し顧客に説明できる |
| G4 | 高難度の税務判断を自ら行い、結果に責任を持つ。チームの専門知識の底上げを意識的に行う |
| G5 | 事務所全体の専門水準を引き上げる役割を担う。新領域（国際税務・M&A等）の知識習得を率先する |

#### 領域②：業務遂行

| Grade | 行動指標 |
|---|---|
| G1 | 期限を守り、提出前に上司に確認する。作業の進捗を日報等で正確に報告する |
| G2 | 期限が危うくなった時点で上司に先に報告し、対策案を提示する。自分の担当案件の全体スケジュールを管理できる |
| G3 | 複数案件の優先順位を自ら判断し、リソースを配分する。後輩のスケジュール管理を支援する |
| G4 | チーム全体の業務量と品質をモニタリングし、過負荷・品質低下を早期に検知して対処する |
| G5 | 繁忙期の業務分散・効率化を組織的に設計する。業務プロセス改善の提案を所長に行う |

#### 領域③：顧客・対外関係

| Grade | 行動指標 |
|---|---|
| G1 | 顧客からの電話・メールに礼儀正しく対応し、不明な点は上司に確認してから回答する |
| G2 | 担当顧客の基本情報・経営状況を把握し、的確な回答ができる。顧客から直接相談を受けられる関係を築く |
| G3 | 顧客の潜在ニーズを把握し、追加サービスを提案する。顧客満足度を高める工夫をチームに提案する |
| G4 | 顧客との長期的な信頼関係を主体的に構築する。クレームが起きたとき自ら対処し、関係を修復できる |
| G5 | 新規顧客の開拓・紹介に貢献する。地域・業界でのネットワークを活かして事務所のプレゼンスを高める |

#### 領域④：チーム・組織貢献

| Grade | 行動指標 |
|---|---|
| G1 | 分からないことを積極的に質問し、学んだことをメモ・共有する。チームの仕事を積極的に手伝う |
| G2 | G1の後輩に業務手順を教えることができる。自分が知った有用な情報をチームに共有する |
| G3 | G1・G2の育成計画を立て、成長を継続的にフォローする。チームの雰囲気改善に能動的に取り組む |
| G4 | チームメンバーの強み・弱みを把握し、適切な業務配分を行う。1on1を定期的に実施し部下の成長を支援する |
| G5 | 事務所全体の人材育成方針を所長と共に設計・実行する。部門を越えた知識共有の仕組みを作る |

#### 領域⑤：変革・成長志向

| Grade | 行動指標 |
|---|---|
| G1 | 担当業務の改善点に気づいたとき上司に提案する。資格取得・自己研鑽に積極的に取り組む |
| G2 | 業務の非効率な部分を自ら見つけ、改善提案を具体的に行う。新しいツール（AI等）を積極的に試す |
| G3 | チームの業務プロセスを改善する提案をし、実行までリードする。後輩の学習意欲を引き出す関わり方をする |
| G4 | 組織の課題を構造的に分析し、解決策を所長に提言する。業界の変化（AI・税制改正等）を先読みし対応策を考える |
| G5 | 事務所の中長期戦略に関与し、変革をリードする。外部環境の変化を踏まえた組織変革を主導する |

## 5. 導入ロードマップ（御社・55名規模）

```
【Phase 準備（2026年7〜9月）】
  Step 1：所長・幹部でドメイン領域と職階を合意する（2時間のワークショップ）
  Step 2：各職階のハイパフォーマー候補者を2〜3名特定する
  Step 3：インタビュー実施、行動指標のドラフト作成
  所要時間：3ヶ月

【Phase 検証（2026年10〜11月）】
  Step 4：ドラフトをスタッフ代表に見せてフィードバックを収集
  Step 5：管理職に「この基準で評価できるか」を確認・修正
  所要時間：2ヶ月

【Phase 実装（2026年12月〜2027年3月）】
  Step 6：管理職研修（フレームワークの読み方・評価への活用法）
  Step 7：パイロット運用（1チームで試行→全社展開）
  評価制度・育成ロードマップとの接続を完成させる
  所要時間：3〜4ヶ月

【Phase 定着（2027年4月〜）】
  全社での正式運用開始
  半年ごとの見直しサイクルを確立
```

## 6. よくある設計ミスと回避策

| ミス | 影響 | 回避策 |
|---|---|---|
| 業界標準の丸写し | 「自社らしさ」がなく形骸化 | ハイパフォーマーのインタビューから自社の言葉で作る |
| コンピテンシー数が多すぎる | 評価が煩雑・負担過多 | 最初は5〜7領域に絞る |
| 行動指標が抽象的 | 評価者によって解釈がバラバラ | 「観察できる行動」に書き直す |
| 等級間の差が不明確 | 昇格基準が分からない | 「指示を受けてやる→自分で判断してやる→他者に教えてやる」の段差を意識する |
| 作って終わり | 翌年には誰も見ない | 年次評価・1on1・育成計画に必ず紐付ける |
| 経営陣だけで作る | 現場の実態と乖離 | スタッフ代表・ハイパフォーマーを必ず設計に巻き込む |
