## 2. 「機能しない1on1」の典型パターン

### よくある失敗

| 失敗パターン | 症状 | 根本原因 |
|---|---|---|
| **進捗報告会になる** | 「今週の業務は〇〇です」で終わる | 目的設定が「管理」になっている |
| **上司が話し続ける** | 部下が「聞く側」に回る | 「上司が教える場」という誤解 |
| **評価の場になる** | 部下が本音を言えない | 評価と1on1が混同されている |
| **記録がない** | 毎回同じ話をしている | 継続性の設計がない |
| **頻度が不安定** | 忙しいとキャンセルされる | 1on1の優先度が低い |

### 上司と部下の本音調査（パーソル総合研究所）

- **上司の課題1位**：「面談について学ぶ仕組みがない」
- **部下の課題1位**：「面談の効果が感じられない」

→ 双方が「手探り状態」のまま実施されているのが現状。**設計と教育が必要**。

## 3. 正しい1on1の定義と原則

### 3-1. 定義

> 「上司が部下の状態を管理する場」ではなく、**「部下が安心して本音を話し、自分の成長を考える場」**

主役は**部下**。上司の役割は聞くこと・質問すること・支援を申し出ること。

### 3-2. 4つの設計原則

| 原則 | 内容 |
|---|---|
| **評価から切り離す** | 1on1での発言が評価に影響しない、という信頼が必要 |
| **部下が議題を持つ** | 「何を話したいか」は部下が考えてくる（上司が決めない） |
| **記録を残す** | 次回に「前回話したこと」を確認できる状態にする |
| **継続する** | 「忙しくてもキャンセルしない」が信頼の証 |

## 4. 設計：頻度・時間・場所

### 4-1. 推奨設定

| 項目 | 推奨 | 理由 |
|---|---|---|
| **頻度** | 隔週（2週に1回） | 週次は負担が大きく形骸化しやすい。月次は間隔が長すぎる |
| **時間** | 30〜45分 | 60分は長く脱線しやすい。15分は短すぎて表面的になる |
| **場所** | 個室 or オフライン | 周囲に聞こえる環境では本音が出ない |
| **曜日・時間帯** | 固定（例：毎週水曜14時） | 予測可能性が信頼感を生む |
| **リード** | 部下が議題を準備 | 上司ではなく部下が主体 |

### 4-2. 実施頻度の段階設計

```
【導入期（最初の2ヶ月）】
  週次 × 30分
  → 関係構築・信頼の土台を作る期間。最初だけ濃度を高くする

【定着期（3ヶ月目〜）】
  隔週 × 45分
  → 本音が出るようになったらペースを安定させる

【成熟期（半年後〜）】
  隔週 × 30〜45分 + 四半期ごとのキャリア面談（60分）
  → ルーティン化した1on1 + 深掘りのキャリア対話を組み合わせる
```

## 5. 議題設計：何を話すか

### 5-1. 禁止事項（これは1on1ではない）

- 業務進捗の確認・報告
- 評価・査定の話
- 上司からの一方的な指示・説明
- 会議の延長線

### 5-2. 推奨テーマ構造（30〜45分）

```
【Opening：5分】
  アイスブレーク / 体調・気分の確認
  「今日はどんな気分ですか？」「最近どうですか？」

【本題①：近況・コンディション（10分）】
  今困っていること・モヤモヤしていること
  「最近、仕事で難しいと感じていることはありますか？」
  「今週、うまくいったことを1つ教えてください」

【本題②：成長・キャリア（10分）】
  学びたいこと・伸ばしたいスキル・将来の方向性
  「今の仕事で最もやりがいを感じる部分は何ですか？」
  「どんなスキルをもっと伸ばしたいですか？」

【本題③：支援・フィードバック（10分）】
  上司やチームへの要望・改善提案
  「私（上司）に何か変えてほしいことはありますか？」
  「チームの仕事の進め方で、改善できそうなことはありますか？」

【Closing：5分】
  次回までのアクション確認
  「次回まで、お互いに何をやっておきますか？」
```

### 5-3. 質問バンク（場面別）

**コンディション・本音を引き出す質問：**
```
- 最近、いつ一番ストレスを感じましたか？その原因は？
- 今の仕事で、もっと時間をかけたいと思うことは？
- 「こんな仕事は自分に向いていない」と感じた瞬間はありましたか？
- 今の職場環境で、変えたいと思うことはありますか？
- 仕事以外で最近うまくいっていることはありますか？
```

**成長・キャリアの質問：**
```
- 1年後、どんなことができるようになっていたいですか？
- 今の仕事で「これは自分の強みだ」と感じる場面はいつですか？
- キャリアについて、最近考えていることはありますか？
- 尊敬している人（社内外問わず）はいますか？その人のどんなところを？
- 今の仕事が将来の自分にどうつながると思いますか？
```

**上司への率直なフィードバックを引き出す質問：**
```
- 私（上司）のサポートで、もっとこうしてほしいということはありますか？
- 私が変えたら、あなたの仕事がやりやすくなることはありますか？
- チームの雰囲気について、正直にどう思いますか？
- 何か、言いにくいと思っていることはありますか？
```

**心理的安全性を高める問いかけ：**
```
- （ミスが起きたとき）何が起きたか、どう感じているか聞かせてもらえますか？
- 誰でも失敗する。次にどうすればいいか一緒に考えましょう
- その提案、もう少し詳しく聞かせてください（即否定しない）
```

## 6. 記録の設計

### 6-1. 記録が必要な理由

記録がないと：
- 「言いっぱなし」になり、部下が「何も変わらない」と感じる
- 成長の変化が見えない（1年前に言っていたことを忘れる）
- 上司が変わったときに引継ぎができない

### 6-2. 記録の4項目

```
【1on1 記録テンプレート】
日付：____年__月__日
参加者：____（上司）/ ____（部下）

① 話したこと（キーワード・トピック）：

② 部下のコンディション（1〜5段階）：
  └ 1（かなり辛い）〜 5（とても好調）

③ アクション（誰が・何を・いつまでに）：
  上司：
  部下：

④ 次回に持ち越すテーマ：
```

### 6-3. 記録ツールの選択肢

| ツール | 特徴 | 向いているケース |
|---|---|---|
| Google Docs / Notion | 柔軟・無料・共有しやすい | 小規模チーム |
| Asana / Trello | タスク管理と連動 | アクション管理を重視する場合 |
| HiManager / kakeai（国産） | 1on1専用・日本語対応 | 1on1に特化したい場合 |
| Excel / スプレッドシート | シンプル・導入障壁低い | まず試したい段階 |

**推奨（御社規模）：** 最初はGoogle Docsの共有ドキュメントで十分。ツールより「記録する習慣」が先。

## 7. 効果測定の設計

### 7-1. 測定すべき3つの層

```
【定量指標】数値で追う
  - 1on1の実施率（予定回数 / 実施回数）
  - 1on1後の満足度スコア（部下が5段階で回答）
  - 離職率の推移（月次）
  - エンゲージメントスコア（四半期ごとのサーベイ）

【定性指標】変化を言語化する
  - 「本音が言えるようになった」という部下の自己申告
  - 1on1後の問題報告スピードの変化
  - 自発的な提案・改善の増加

【関係性指標】関係の深まりを測る
  - 部下から上司への自発的な相談件数
  - 1on1以外での声かけの頻度変化
```

### 7-2. 部下向け効果測定サーベイ（四半期ごと・匿名）

```
以下を5段階（1：全くそう思わない〜5：強くそう思う）で評価してください：

1. 上司との1on1では、本音を話せていると感じる
2. 1on1の時間は、自分にとって有意義だと感じる
3. 1on1で話したことが、実際に行動・変化につながっている
4. 上司は私の話をきちんと聞いてくれていると感じる
5. 1on1の後、仕事へのモチベーションが上がることが多い

自由記述：1on1について改善してほしいことがあれば書いてください。
```

### 7-3. KPIと目標値（導入後6ヶ月）

| KPI | 目標値 | 測定方法 |
|---|---|---|
| 1on1実施率 | 90%以上 | 実施記録 |
| 部下満足度スコア | 4.0以上（5段階） | 四半期サーベイ |
| エンゲージメントスコア変化 | +15ポイント以上 | 半期比較 |
| 離職率変化 | 前年比▲30%以上 | 人事記録（半年後〜1年で効果発現） |

## 8. 上司のスキル育成：1on1がうまくできない理由

「制度を作っても上司が下手だと機能しない」が最大の落とし穴。

### 8-1. 上司が陥りやすい行動パターン

| NG行動 | 影響 | 代替行動 |
|---|---|---|
| 解決策をすぐ出す | 部下が「考えてもらえない」と感じる | 「あなたはどう思う？」と質問する |
| 話を遮る | 部下が「聞いてもらえない」と感じる | 最後まで聞いてから話す |
| スマホ・PCを見る | 「自分は重要じゃない」と感じる | 1on1中はデバイスを閉じる |
| アドバイスしすぎる | 部下の自律性が育たない | 「一緒に考える」スタンスを持つ |
| 評価の話をする | 本音が出なくなる | 「今日は評価の話は関係ない」と明示する |

### 8-2. 上司研修の設計（3ステップ）

**Step 1：知識インプット（1回・2時間）**
- 1on1の目的と原則
- 聞く技術（アクティブリスニング）の基本
- 良い質問・悪い質問の違い

**Step 2：ロールプレイ（1回・2時間）**
- ペアで上司・部下を演じる実践練習
- 「今の会話の何が良くて何が悪かったか」のフィードバック

**Step 3：継続フィードバック（月次・30分）**
- 各上司が「今月うまくいかなかった1on1」を1件持ち寄る
- チームで改善策を議論する（上司同士の学びあい）

## 9. 導入ロードマップ（御社・55名規模）

### Phase 0（導入前準備・1ヶ月）

| タスク | 担当 | 内容 |
|---|---|---|
| 目的の明文化と全社共有 | 所長 | 「なぜ1on1をやるのか」を朝礼・文書で伝える |
| 上司研修の実施 | 人事・外部講師 | Step 1〜2の研修（3〜4名の上司・シニア全員対象） |
| 記録テンプレートの準備 | 人事 | Google Docs等でテンプレートを用意・共有 |
| スケジュール確保 | 全員 | 隔週の1on1をカレンダーに先行登録 |

### Phase 1（導入期・1〜2ヶ月）

- 週次 × 30分でスタート（信頼構築期）
- 上司は「聞く・質問する」に徹する
- 記録は簡易（4項目）から始める
- 所長は全員の1on1実施率を週次で把握する

### Phase 2（定着期・3〜6ヶ月）

- 隔週 × 45分に移行
- 四半期ごとに「部下満足度サーベイ」を実施
- 上司の月次フィードバック会を開始
- エンゲージメントサーベイと連動させる

### Phase 3（成熟期・6ヶ月〜）

- 隔週ルーティン + 四半期キャリア面談（60分）の二層構造
- 離職率・エンゲージメントスコアの前年比較を定期報告
- 優れた1on1事例を社内で共有（ベストプラクティスの横展開）

## 10. 成功事例スナップショット

### 事例① ヤフー株式会社（週次1on1の先駆者）
- 全マネージャーに週次1on1を義務化
- 上司研修・1on1コーチングを継続実施
- **効果：** エンゲージメントスコア継続向上、定着率改善

### 事例② テモナCSチーム
- 1on1を正式制度として導入
- **効果：** 1年で離職率 **20% → 0%** に低下

### 事例③ トランスコスモス（2024年）
- 1on1と社内共有会（パーパスの伝達）を組み合わせ
- 「自分の仕事が組織の成長に直結している」と感じる社員が増加
- **効果：** 従業員満足度 **58% → 78%**

### 事例④ GitLab / HubSpot（2025年グローバルトレンド）
- 「部下が議題を決め、部下が会議をリードする」形式に移行
- **効果：** 部下の主体性・自律性が向上。上司への信頼感が増加

## 11. 税理士法人固有の設計考慮点

| 課題 | 設計上の対応 |
|---|---|
| **繁忙期はキャンセルしがち** | 繁忙期（1〜3月）は隔週→月次に変更OKと予め明示 |
| **資格勉強中の若手は時間がない** | 30分以内に厳守。勉強時間を侵食しない |
| **評価する/される関係が強い** | 初回1on1で「この時間は評価に関係しない」と明言する |
| **話す習慣がない職人気質のシニア** | シニア対象の1on1は「コーチ型」でなく「相談型」から入る |
| **所長が全員と話すのは難しい** | 所長→シニア、シニア→若手の2段階構造で設計する |
