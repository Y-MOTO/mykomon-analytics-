**対象：所長**　　**最終更新：2026-04-25（日報分析連携機能・マニュアルボタン追加）**

## このアプリの意義

### なぜ「ずれの認識」が経営の核心か

経営改革が失敗する最大の原因は、**計画を立てたにもかかわらず現実とのずれを放置すること**です。ずれは必ず生じます。問題はずれの存在ではなく、ずれに気づかないこと・気づいても対処できないことにあります。

人事制度改革は特にこのリスクが高い領域です。職員の行動・感情・文化が絡むため、ずれの原因が複合的で、所長一人の経験と直感だけでは根本原因を特定しにくいのです。

### このアプリが解決する問題

| 問題 | このアプリによる解決 |
|---|---|
| 「うまくいかない」と感じても原因がわからない | 文化的・制度的・スキル的に分類して根本原因を仮説立てる |
| 外部コンサルタントに相談する時間・コストがない | いつでも・無制限に・数円/回で相談できる |
| 日本国内の事例しか知らない | G8各国・世界の成功・失敗事例と比較できる |
| 人事の一般知識が不足している | 経営者視点の人事理論・採用・定着・世代論を提供する |
| 日報データを見ても次の行動に結びつかない | 集計数値を入力として根拠ある改善策を提案する |

### 日報分析アプリとの役割分担

```
日報分析アプリ          人事経営相談アプリ
「現状を見る」    ＋    「対策を考える」
     ↓                      ↓
 数値・グラフ          原因分析・改善策
（何が起きているか）   （なぜか・どうするか）
```

2つのアプリはセットで使うことで、**データに基づく経営判断サイクル**が回ります。

### 相談できる内容の範囲

このアプリは「改革計画のずれ」だけでなく、**経営者としての人事全般の悩み**にも対応します。

- 人事制度改革の進捗に関するずれ（Phase 1〜3）
- G8各国・世界の事務所がどう考え・どう動いているかの比較研究
- 採用・定着・育成・評価に関する経営者の一般的な悩み
- Z世代・ベテラン層など世代間マネジメントの課題
- 心理的安全性・ボトムアップ文化の醸成方法

## このアプリでできること

「計画と現実のずれ」や「経営者としての人事の悩み」を入力すると、Claude AIが以下を提案します。

- 課題の分類（文化的・制度的・スキル的・複合的）
- 根本原因の仮説
- 今週すぐに取れる対策（コストゼロ）
- 1〜3ヶ月の中期対策
- G8各国・世界の参考事例と御社への示唆

AIは御社の会社規模・人事制度改革計画・現在のフェーズ（Phase 1）を踏まえた上で回答します。

## 起動方法

**`【ローカル版】【人事経営相談】起動.bat`** をダブルクリックします。

数秒後、ブラウザが自動で開きます。

```
http://localhost:8502
```

> 日報分析アプリ（8501番）と同時に起動できます。両方開いたまま使用可能です。

### アプリが起動しない場合

コマンドプロンプトで以下を実行してください。

```
cd /d "（日報分析アプリフォルダのパス）"
python -m streamlit run consult_app.py --server.port 8502
```

## 基本的な使い方（4ステップ）

### ステップ1　カテゴリを選ぶ（任意）

左サイドバーの「📂 相談カテゴリ」で相談テーマを選択します。

| カテゴリ | 使う場面 |
|---|---|
| （自動判断） | AIに任せる（迷ったらこれ） |
| ボトムアップ・自主経営 | 発言が出ない、提案がない |
| 業務の見える化・日報 | タグ付き率が上がらない |
| 評価制度・人事考課 | 評価への不満・不信 |
| 給与・報酬体系 | 給与設計・ボーナス |
| 職階・役割分担 | TierやOKRの運用 |
| 有資格者の処遇 | 資格者からの不満 |
| 職員の教育・育成 | 後輩指導・スキル不足 |
| 採用・離職 | 採用難・離職増加 |
| 顧客対応・サービス品質 | 顧客からのクレーム |
| その他 | 上記に当てはまらない |

### ステップ2　相談内容を入力する

「計画と現実のずれ・困っていること」の入力欄に、起きていることを自由に書きます。

**例文：**

> ボトムアップ会議を始めて2ヶ月たつが、誰も発言しない。どう対処すればよいか。

#### クイック入力ボタンを使う

入力欄の上の「💡 相談例」をクリックすると、よくある相談例の一覧が開きます。  
ボタンをクリックすると入力欄に自動入力されます（修正してから送信できます）。

**相談例一覧：**

- ボトムアップ会議を始めたが誰も発言しない
- 日報のタグ付き率が20%で上がらない
- 評価制度を変えたら古参職員が不満を持っている
- 有資格者が自分だけが評価されていないと言っている
- 後輩への指導が減って業務品質が下がっている
- OKRを設定したが形骸化している
- 音声入力を使わない職員がいる
- 集合ボーナスの目標設定をどうすればよいか

### ステップ3　「相談する」ボタンを押す

「📨 相談する」ボタンを押すと分析が始まります（10〜30秒）。

AIは以下の構成で回答します。

1. **課題の分類** — 文化的課題・制度的課題・スキル的課題などに分類
2. **根本原因の仮説** — 御社の状況と照合した分析
3. **今週できる対策** — コストゼロですぐ動ける施策
4. **1〜3ヶ月の対策** — Phase 1の計画と整合した中期施策
5. **参考事例** — G8各国・世界の類似事例

### ステップ4　フォローアップ質問で深掘りする

回答の下に「フォローアップ質問」欄が現れます。  
そのまま続けて質問すると、会話形式で深掘りできます。

**フォローアップ例：**

- 「具体的にどのような言葉で職員に説明すればよいですか？」
- 「Buurtzorgの事例をもっと詳しく教えてください」
- 「Phase 2に進む前に確認しておくべきことは何ですか？」

## サイドバーの機能

### 📖 使用マニュアル

このマニュアル自体をいつでも参照・印刷できます。

| ボタン | 動作 |
|---|---|
| **📄 表示** | ブラウザでマニュアルを表示（見出し・表を整形） |
| **🖨️ 印刷** | ブラウザの印刷ダイアログを起動 |

### 📊 日報分析データを使う

日報分析アプリの集計結果を相談の「ズレ」として直接活用できます。  
詳しくは「[日報分析データとの連携](#日報分析データとの連携)」を参照してください。

| 表示内容 | 説明 |
|---|---|
| 期間・取得日時 | 日報分析アプリから送られたデータの対象期間 |
| 全件数 | 分析対象の全日報件数 |
| タグ付き率 | 拡張機能使用率（Phase 1 の目標は 80%） |
| 📥 この結果を相談に使う | 集計数値を入力欄に自動展開する |

> データがない場合は「日報分析アプリのタブ5で「📨 人事経営相談に送る」を押してください」と表示されます。

### 📋 現在の計画概要

Phase 1〜3の概要を確認できます。相談中の「現在地」の確認に使います。

### 🗑️ 今回の会話をリセット

現在の会話履歴をクリアします。新しいテーマで相談する際に使います。  
※過去の履歴（右パネル）には影響しません。

## 相談履歴の活用

### 自動保存

相談内容と回答は `consult_history.json` に自動保存されます。  
アプリを閉じても履歴は残ります。

### 過去の相談を確認する

画面右の「🗂️ 過去の相談履歴」パネルに最新20件が一覧表示されます。  
タイトル行をクリックすると展開して回答全文を読めます。

### JSONでダウンロード

「📥 履歴をJSONで保存」→「ダウンロード」で全履歴をJSONファイルとして保存できます。  
月次ミーティングの資料整理や制度見直しの参考資料として活用できます。

### 履歴の全削除

「🗑️ 履歴を全削除」で全件削除できます（元に戻せません）。

## 日報分析データとの連携

日報分析アプリ（ポート8501）で蓄積した集計数値を、そのまま人事経営相談の「ズレ」として活用できます。

### 操作手順

**① 日報分析アプリ側（タブ5 AI分析レポート）**

1. 「📝 AIレポートを生成」でレポートを生成する
2. 画面下部の「**📨 人事経営相談に送る**」ボタンを押す
3. 「✅ 人事経営相談アプリにデータを送りました」と表示されれば完了

**② 人事経営相談アプリ側（サイドバー）**

1. サイドバーの「📊 日報分析データを使う」に期間と数値が表示される
2. 「**📥 この結果を相談に使う**」を押す
3. 相談入力欄に以下の形式でデータが自動展開される

```
【日報分析データ 2026年1月〜4月】（2026-04-25 10:30 取得）
全日報件数：8,046件　タグ付き：312件（3.9%）　詰まり：87件

■ タグ付き率が低い担当者（上位5名）
・田中：0.0%
・鈴木：5.2%

■ 主な阻害要因
1位：資料待ち（32件）
2位：確認中（25件）

---
【所長の補足・感じているずれ（任意）】
（ここに追記してください）  ← 所長が主観・背景を追記してから送信
```

4. 必要に応じて「所長の補足」欄に感じていることを追記する
5. 「📨 相談する」を押す

### データの特性と使い分け

| 情報の種類 | 内容 | 入力元 |
|---|---|---|
| **客観データ**（自動） | 件数・タグ付き率・阻害要因 | 日報分析アプリから自動転記 |
| **主観・文脈**（手動） | 職員の雰囲気・背景事情 | 所長が補足欄に追記 |
| **改革計画**（固定） | Phase目標・制度設計・失敗パターン | knowledge.py（常時参照） |

この3層構造により、AIが最も精度高く原因分析と対策提案を行えます。

## よくある問題

### 「APIキーを入力してください」と表示される

サイドバー上部の「Anthropic API キー」欄にAPIキーを貼り付けてください。  
通常は環境変数で自動設定されます。消えている場合は **`APIキー設定.bat`** を実行してアプリを再起動してください。

### 「エラー: ...」と表示されて回答が出ない

- Anthropicのアカウントに残高があるか確認してください
- インターネット接続を確認してください

### ブラウザが開いても画面が真っ白

数秒待ってからページを更新（F5）してください。  
解消しない場合はコマンドプロンプトのエラーメッセージを確認してください。
